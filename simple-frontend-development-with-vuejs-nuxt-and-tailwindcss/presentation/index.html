  • CSS schreiben & maintainen
    • Mehr Komponenten -> Mehr CSS
    • Größere CSS Datei -> längere Downloadzeiten
    • schon mal ein 30k Lines großes CSS maintained?
  • Klassen(namen) ausdenken
    • Keine Reusability
    • Wenn man es reusable machen will, muss man ordentlich Gehirnschmalz reinstecken
    • Bei Änderung: Gefahr andere Komponenten kaputt zu machen
Tailwind to the rescue
Vordefinierte Utility Klassen statt Komponenten

Vorteile
  • Kein CSS selber schreiben
  • Keine Klassen(namen) mehr ausdenken müssen
  • Änderungen betreffen nur die ausgewählte Komponente, nicht automatisch alle anderen
